# Nimora 设计美学与体验规范

> 版本：0.1.0-draft  
> 更新日期：2026-07-17  
> 适用范围：桌宠窗口、控制中心、命令面板、气泡、通知、商店、Creator Studio

## 1. 设计愿景

Nimora 的视觉目标不是“把工具做得可爱”，而是让一个有生命感的角色自然存在于桌面，同时让复杂的平台能力保持安静、可信和可理解。

设计关键词：

- **有生命感**：动作有动机，反馈有情绪，等待也有细节。
- **轻盈**：不遮挡工作，不制造视觉噪声，不持续争夺注意力。
- **温暖**：语言、声音、动效和色彩让用户感到被陪伴，而不是被管理。
- **克制**：功能复杂度藏在渐进式界面中，默认体验保持简单。
- **可信**：权限、AI 行动和数据出站永远清楚、可预测、可撤销。
- **可创作**：官方设计系统提供约束，但不压制角色和主题的独特风格。

## 2. 体验层级

```text
第一层：宠物本体——情绪和陪伴
第二层：即时交互——气泡、菜单、提醒
第三层：快捷效率——命令面板、任务和自动化
第四层：深度配置——控制中心、信任中心、Creator Studio
```

用户不应为了完成第一层体验而进入第四层。复杂界面不能侵入桌面陪伴状态。

## 3. 品牌视觉方向

### 3.1 官方基础风格

官方 UI 采用“柔和数字工作台”风格：

- 低饱和中性色作为大面积背景。
- 角色强调色只用于当前状态、主要操作和轻量装饰。
- 圆角体现亲和力，但不使用过度胶囊化布局。
- 阴影用于表达层级，不用于装饰性发光。
- 图标保持清晰和几何一致，不使用表情符号代替功能图标。

### 3.2 品牌与角色分离

平台品牌色与角色主题色必须分离：

- 平台色表达系统状态、权限和操作层级。
- 角色色表达人格、情绪和世界观。
- Theme Pack 可以覆盖装饰变量，但不能改变危险、错误、警告和成功的语义色。

## 4. Design Token

所有 UI 使用语义 Token，不直接引用固定颜色或尺寸。

```css
:root {
  --dp-color-surface-base: #f6f4f1;
  --dp-color-surface-raised: #ffffff;
  --dp-color-text-primary: #252321;
  --dp-color-text-secondary: #6e6963;
  --dp-color-accent: #7b74e8;
  --dp-color-success: #2f8f63;
  --dp-color-warning: #b56b13;
  --dp-color-danger: #c84747;
  --dp-radius-sm: 8px;
  --dp-radius-md: 12px;
  --dp-radius-lg: 18px;
  --dp-space-1: 4px;
  --dp-space-2: 8px;
  --dp-space-3: 12px;
  --dp-space-4: 16px;
  --dp-space-6: 24px;
  --dp-motion-fast: 120ms;
  --dp-motion-normal: 200ms;
  --dp-motion-slow: 320ms;
}
```

Theme Pack 只能覆盖公开 Token。平台必须校验文本对比度和状态色可辨识度。

## 5. 色彩规范

| 语义 | 用途 | 禁止 |
|---|---|---|
| Accent | 主操作、选中、焦点 | 大面积背景持续高饱和 |
| Success | 完成、连接健康 | 表示普通启用状态 |
| Warning | 权限扩大、性能异常 | 与 Danger 混用 |
| Danger | 删除、外部暴露、高风险执行 | 用于营销或装饰 |
| Info | 中性说明、新能力 | 替代主要层级结构 |

- 正文与背景对比度至少满足 WCAG AA。
- 信息不能仅靠颜色表达，必须结合文字、图标或形状。
- 深色主题不是简单颜色反转，要重新校验阴影、透明度和角色边缘。

## 6. 字体与排版

- 使用系统 UI 字体，中文优先匹配系统黑体。
- 默认正文 14 px，辅助信息不低于 12 px。
- 标题层级控制在 3 级以内，避免控制中心像文档编辑器。
- 数字、端口、事件 ID 和代码使用等宽字体。
- 单行文本按钮使用动词开头，例如“启用连接器”“查看发送内容”。
- 危险确认必须写清动作和影响，不使用模糊的“确定吗”。

## 7. 布局与密度

### 7.1 控制中心

- 使用侧边一级导航和页面内分组。
- 默认内容宽度 720–960 px，避免超宽表单。
- 设置遵循“标题 → 一句话说明 → 控件 → 状态反馈”。
- 高级配置折叠展示，并显示恢复默认值入口。
- 长列表支持搜索、过滤和状态摘要。

### 7.2 桌宠浮层

- 气泡宽度随内容变化，但不超过当前显示器可用宽度的 40%。
- 气泡优先避开活动窗口重要区域和屏幕边缘。
- 桌宠菜单不复制完整控制中心，只保留高频即时操作。
- 点击穿透开启后，必须始终存在托盘或热键恢复路径。

### 7.3 响应式

控制中心最低有效宽度为 720 px。窄窗口将双栏设置折叠为单栏，不缩小文字或点击区域。

## 8. 宠物视觉与动作美学

### 8.1 动作原则

- **准备—动作—恢复**：重要动作包含 anticipation、action、settle 三段。
- **重量感**：拖拽、落地和转身需要惯性，不瞬间切换位置。
- **目光关系**：宠物看向气泡、鼠标或目标时保持方向一致。
- **节奏变化**：Idle 包含长短变化，避免机械循环。
- **少即是多**：桌面工作场景下，不持续播放大幅动作。

### 8.2 动作优先级

```text
安全与系统反馈
  > 用户直接交互
  > 任务和提醒演出
  > 状态表达
  > 自主 Idle
```

高优先级动作打断低优先级动作时，应该使用过渡 Clip 或安全落点。

### 8.3 情绪表达

情绪不能只靠气泡文字表达，应组合：

- 姿态与速度。
- 眼睛、耳朵、尾巴等角色特征。
- 粒子和辅助图形。
- 音效或触觉替代提示。
- 气泡语气和停留时间。

不同 Character 可以有不同表达方式，但必须映射到统一情绪语义。

## 9. 动效系统

| 场景 | 推荐时长 | 曲线 |
|---|---:|---|
| Hover/Press | 80–120 ms | ease-out |
| 面板展开 | 160–220 ms | standard |
| 页面切换 | 180–260 ms | standard |
| 成功反馈 | 240–400 ms | emphasized |
| 宠物位移 | 按距离计算 | physical/ease-in-out |

- UI 动效不阻塞输入。
- 同时出现的动画焦点不超过两个。
- `prefers-reduced-motion` 下取消位移、缩放和闪烁，保留淡入淡出。
- 任务执行中的循环动效必须有静态状态文本。

## 10. 气泡与文案语气

### 10.1 气泡类型

| 类型 | 特征 | 示例 |
|---|---|---|
| Ambient | 无按钮、短暂、可忽略 | “今天也在这里陪你。” |
| Reminder | 1–3 个明确操作 | “坐了 45 分钟，要活动一下吗？” |
| Task | 显示进度、结果和取消 | “正在切换到专注状态…” |
| Permission | 清晰说明主体、数据和目标 | “日历技能想读取未来 7 天日程。” |
| Error | 解释影响和恢复动作 | “皮肤加载失败，已恢复默认角色。” |

### 10.2 文案规则

- 不责备用户，不使用羞辱式打卡文案。
- 不伪装成人类，不声称拥有真实感受或现实经历。
- 主动建议说明原因，例如“你已经连续工作 50 分钟”。
- 错误提示包含发生了什么、当前影响和下一步。
- Agent 执行完成后说明实际执行结果，不只说“好了”。

## 11. 声音设计

- 声音默认克制，连续相似提示需要冷却。
- 音效分为互动、状态、提醒、成功、警告五类。
- 警告音不能被 Theme Pack 替换为含糊或愉悦音效。
- 所有声音可独立关闭，并提供字幕或视觉替代。
- 夜间 Profile 默认降低音量或静音。

## 12. 权限与信任界面

权限页面必须回答：

1. 谁在请求？
2. 想完成什么？
3. 将访问哪些数据或系统能力？
4. 将数据发送到哪里？
5. 允许多久？
6. 如何撤销？

危险操作使用清晰的红色语义区域，但避免制造无意义恐慌。批量授权不得预选高危选项。

## 13. AI 交互设计

- 明确区分宠物说话、Agent 建议和系统状态。
- Agent 计划使用步骤列表，展示实际工具和关键参数。
- 用户可以修改、取消、单步批准或保存为自动化。
- 执行中显示当前步骤；失败时保留已完成步骤和回滚状态。
- 不使用“AI 正在思考”掩盖网络超时，应显示可取消的真实状态。

## 14. Creator Studio 美学规则

- 编辑器以画布为中心，属性面板为辅助。
- 时间轴、命中区和状态图使用一致的选择与颜色语义。
- 预览背景可切换桌面、浅色、深色和透明棋盘格。
- 性能、缺失动作、许可证和可访问性问题在发布前集中展示。
- 创作者可以覆盖角色美学，但不能覆盖安全、权限和系统错误语义。

## 声音美学与安静体验

- 声音是角色表现层，不是系统安全反馈层；第三方音色不得模拟或替换权限、危险、错误和恢复提示。
- 安装前音频预览同时展示 Cue、格式、字节大小、增益与本地化字幕，让用户在播放前理解内容。
- 音频控件保持平台熟悉语义并支持键盘操作，不自动播放；预览卡片明确标注本地验证、无代码、无网络。
- Quiet Mode 是可靠的用户承诺：开启后不请求活动 Clip、不创建 Audio 对象，也不因静音改变动作执行结果。
- 声音增益由 dB 转换为浏览器线性音量并钳制到 `0..1`；播放结束或失败立即释放 Blob URL。

## 14.1 用户模式与信息密度

| 模式 | 美学重点 | 默认隐藏内容 |
|---|---|---|
| Companion | 安静、温暖、零配置、角色留白 | Schema、日志、渲染参数和权限细节摘要之外的信息 |
| Character | 角色舞台、收藏、换装与表情表现 | 开发工具与底层事件流 |
| Power User | 快捷、可扫描、状态密度高 | 无；使用渐进展开避免长期噪声 |
| Creator | 画布、时间轴、映射与实时预览 | 与当前创作任务无关的运维面板 |
| Developer | 等宽数据、契约、Trace、性能和调试 | 无，但危险能力仍使用产品化权限界面 |

- 模式切换不改变数据、权限或功能可用性，只改变导航、默认布局和解释深度。
- 二次元角色体验允许鲜明演出与主题化 UI，但系统错误、权限、离线和危险操作保持稳定语义。
- 极客和开发者可提高信息密度、关闭装饰动效并固定诊断面板；不得以“高级模式”绕过确认。
- 普通用户导入第三方模型时使用向导和自动修复，高级报告可随时展开且不隐藏许可证风险。
- Creator Studio 可保存工作区布局；控制中心与桌宠浮层仍遵守无障碍、恢复路径和视觉层级基线。

## 15. 设计验收清单

- 新用户无需进入设置即可完成首次互动。
- 桌宠不会遮挡恢复点击穿透或退出应用的路径。
- 深浅主题、200% 缩放和减少动画模式可用。
- 所有危险操作说明主体、动作、目标和后果。
- 角色资源切换后平台 UI 仍保持清晰和一致。
- 宠物动作具有过渡，不出现高频闪烁和机械抖动。
- 空状态、加载、失败、离线和权限拒绝状态均有设计。
- 主题安装前预览必须局部隔离，并同时展示九色语义色板、深浅模式、圆角密度和动态偏好。
- 深色主题必须逐页验证文本、边框、浮层和危险态对比度；禁止用简单反色或让装饰色覆盖安全语义。
- Theme 自动准入只保证最低静态门槛，不替代 200% 缩放、色觉差异、截图和读屏人工审查；正文最低 `4.5:1`，UI 语义色最低 `3:1`。
