# Nimora 可靠性、鲁棒性与稳定性规范

> 版本：0.1.0-draft  
> 更新日期：2026-07-17  
> 本文定义运行时稳定性和故障恢复的发布门禁。

## 1. 可靠性目标

| 指标 | 目标 |
|---|---|
| Core 会话无崩溃率 | ≥ 99.9% |
| 24 小时连续运行 | 无崩溃、死锁、句柄泄漏 |
| 空闲内存增长 | 24 小时 < 10% |
| UI 崩溃恢复 | 5 秒内恢复可交互 |
| 扩展故障隔离 | 不终止 Core，不损坏共享状态 |
| 安全模式切换 | 2 秒内停止外联和第三方执行 |
| 配置写入 | 原子、可校验、可回滚 |

## 2. 稳定性设计原则

- Core 不等待非关键网络请求和扩展回调。
- 所有跨进程、网络和文件操作都有超时、取消和大小限制。
- 每个队列都有容量、背压和溢出策略，禁止无限增长。
- 所有持久化更新采用事务、临时文件或 write-rename 模式。
- 所有后台任务由统一 Supervisor 管理，禁止失控 detached task。
- 系统时钟跳变、休眠恢复、网络切换和显示器变化是正常事件，不是异常。

## 3. 故障域

| 故障域 | 允许影响 | 不允许影响 |
|---|---|---|
| Asset | 当前资源预览或切换 | Core、用户配置、上一可用资源 |
| WebView | 当前 UI 会话 | Pet 状态、自动化调度、持久化 |
| Skill | 自身贡献和任务 | 其他 Skill、Core、Gateway |
| Connector | 自身投递 | Event Bus 和其他 Connector |
| Agent | 当前 Agent 任务 | Command Registry 和本地自动化 |
| Storage | 相关写操作 | 覆盖最后可用快照 |
| Platform Adapter | 对应平台能力 | 整个应用无提示退出 |

## 4. Supervisor 模型

每个长期运行组件必须声明：

- 启动依赖和 readiness 条件。
- 心跳或健康检查。
- 停止超时和强制终止策略。
- 重启策略、最大次数和退避。
- 崩溃后是否可安全恢复。
- 配额和异常指标。

连续崩溃采用滑动窗口：5 分钟内 3 次失败进入 quarantine。用户手动恢复前不得无限重启。

## 5. 超时、重试和幂等

- 重试只用于瞬时故障，配置错误和权限拒绝不重试。
- 指数退避必须加入 jitter，防止同步重连。
- Command 默认不重试；只有声明 `idempotent` 或提供 idempotency key 才可重试。
- Event ID 表示事实身份，Delivery ID 表示一次投递尝试。
- 超时必须取消下游工作；不能只停止等待而留下后台副作用。

## 6. 队列与背压

| 通道 | 默认策略 |
|---|---|
| UI 状态更新 | 合并同类最新值 |
| Pet 高频移动事件 | 采样或 debounce |
| 审计事件 | 有界磁盘缓冲，Restricted 数据不落正文 |
| Connector 投递 | 每连接器独立有界队列 |
| Automation 实例 | 按规则设置 drop/queue/cancel_previous |
| Agent Tool | 串行或显式并发上限 |

Event Bus 不保证无限保留。订阅者处理缓慢时必须隔离，不得阻塞发布者。

## 7. 持久化鲁棒性

- 数据库启用事务和完整性检查。
- 配置保存前校验 Schema，保存后重新读取校验 hash。
- 升级前创建带版本的快照。
- 启动时检测未完成迁移和半写入文件。
- 磁盘空间不足时停止非关键日志和缓存，保护配置与用户数据。
- 数据修复工具默认只读扫描，修改前生成备份。

## 8. 休眠、恢复与时间

- 使用单调时钟计算间隔，墙上时钟用于日历语义。
- 系统休眠期间不补跑所有错过的 interval；规则声明 `skip`、`run_once` 或 `catch_up`。
- 时区变化重新计算 cron，但不重复执行已完成实例。
- 恢复后分批重连，避免网络和 CPU 峰值。
- 动画时间差设置上限，避免恢复后物理状态跳跃。

## 9. 资源与性能保护

- 纹理、音频、日志、事件历史和 Agent 会话均设置上限。
- 不可见宠物降低帧率或暂停渲染。
- UI 长列表使用虚拟化。
- 扩展使用独立 CPU、内存、定时器和消息预算。
- 性能降级顺序：粒子 → 阴影 → 动画帧率 → 非活跃纹理，而不是停止核心交互。

## 10. 混沌与长稳测试

必须注入：进程崩溃、磁盘满、配置损坏、数据库锁、网络抖动、DNS 失败、系统休眠、时钟跳变、扩展事件风暴、WebView 重载和资源包损坏。

每个稳定版本执行：

- 24 小时日常负载 soak。
- 72 小时低频后台 soak。
- 1 小时事件风暴与限流测试。
- 100 次启动/退出循环。
- 50 次资源和 Profile 热切换。

## 11. 恢复等级

```text
L0 自动恢复：瞬时网络、UI 重载
L1 功能降级：停用单扩展或连接器
L2 安全模式：停止所有非核心能力
L3 只读恢复：数据库或迁移异常
L4 用户数据恢复：从快照、备份或导出恢复
```

恢复必须明确展示当前等级、受影响能力和下一步，不允许静默丢弃用户数据。
