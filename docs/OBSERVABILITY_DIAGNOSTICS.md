# Nimora 可观测性与诊断规范

> 版本：0.1.0-draft  
> 更新日期：2026-07-17

## 1. 目标

可观测性必须帮助用户和开发者回答：系统当前是否健康、哪个组件失败、为什么触发或未触发、数据去了哪里、是否可以安全恢复。

## 2. 信号

### 2.1 日志

- 结构化日志包含时间、等级、组件、事件名、Trace ID 和错误码。
- 默认不记录对话原文、密钥、完整文件路径和 Restricted 数据。
- 用户界面错误使用稳定错误码，并链接到本地诊断解释。
- 日志按大小和日期轮转，保留期可配置。

### 2.2 指标

| 组件 | 核心指标 |
|---|---|
| Core | 启动时间、事件速率、主循环延迟 |
| Renderer | FPS、纹理内存、掉帧、可见状态 |
| Extensions | CPU、内存、消息量、崩溃、重启 |
| Automation | 触发、跳过、失败、运行时间、队列 |
| Connectors | 连接状态、延迟、重试、丢弃、字节 |
| Gateway | 请求、状态码、Scope 拒绝、连接数 |
| Agent | 任务、步骤、Tool 成功率、Token、费用估算 |

### 2.3 Trace

一次用户或外部意图使用同一 Trace ID 串联：输入 → Command → Event → Automation/Skill → Connector/Agent → 审计。跨进程调用创建 Span，不将 Trace ID 当作权限凭证。

## 3. 健康状态

每个组件报告：`starting`、`ready`、`degraded`、`failed`、`stopped`。控制中心汇总健康状态，但不因一个非核心组件失败将整个系统标为不可用。

## 4. 本地诊断中心

提供：

- 当前版本、平台、WebView、数据 Schema 版本。
- 组件健康、扩展崩溃、连接器和端口。
- 最近错误、规则执行历史和事件 Trace。
- 资源预算和性能趋势。
- 只读配置检查、数据库完整性检查和包校验。
- 一键生成脱敏诊断包。

## 5. 诊断包

诊断包默认包含版本、系统摘要、脱敏日志、Schema 版本、包清单、健康报告和用户选中的 Trace。默认不包含密钥、对话、文件内容、用户名和精确窗口标题。

生成前展示内容预览，用户可以逐项取消。诊断包不会自动上传。

### 当前实现

- Control Center 可先预览再导出 `*.nimora-diagnostics.zip`；安全模式和数据库恢复模式均可使用。
- 当前 `nimora.diagnostic-report/1` 包含应用版本、操作系统/架构、启动与安全模式、Outbox 计数、当前数据库 Schema、备份健康摘要和可选结构化事件数量。
- 用户可在导出前取消最近保留的结构化事件；选中时增加 `events.json`，当前只允许固定的启动、恢复启动和自动备份结果事件码、等级、组件和时间戳，不存在任意消息或键值字段。
- 严格结构化事件持久保存于应用数据目录的私有 JSONL 分段；默认保留 14 天、每段最多 1 MiB、最多 64 个文件，跨重启恢复时跳过符号链接、非法文件名、损坏行和截断尾行。
- 当前进程维护最新 256 条内存快照，`events.json` 具有独立 256 KiB 导出预算；达到条目上限时丢弃最旧事件，磁盘不可用时安全降级为仅内存日志而不阻止正常或恢复模式启动。
- ZIP 始终包含 `report.json` 与带 SHA-256 inventory 的 `manifest.json`，使用临时文件、文件同步和原子发布；拒绝覆盖现有目标。
- 当前包明确不包含自由文本日志、Trace、用户名、用户正文、密钥、数据库内容和文件路径，也不会自动上传。
- 自由文本运行日志、级别/时间筛选、用户选择 Trace、包清单和更多来源逐项取消尚未接入；当前持久事件日志不等同于完整日志系统，每个新来源必须先具备独立脱敏器与大小预算。

## 6. 遥测

- 遥测默认策略必须在产品决策中明确，不能由 SDK 默认开启。
- 即使启用，也只收集聚合性能、崩溃和功能可用性信号。
- 提供本地查看、关闭和清除队列能力。
- 不采集对话原文、文件内容、活动窗口标题和用户自定义文本。
- 离线时使用有界队列，过期数据直接删除。

## 7. 告警和用户提示

- 用户只收到可行动的提示，不为每次自动重试弹窗。
- 重复错误聚合显示。
- 性能降级、权限变化和连接器持续失败进入控制中心状态区。
- Critical 安全和数据风险必须立即提示并建议进入安全模式。

## 8. 调试接口

开发者模式可以查看 Event、Command、IPC 和 Connector Trace，但必须继续执行脱敏。生产版不得暴露任意对象检查、执行表达式或跳过权限的后门。
