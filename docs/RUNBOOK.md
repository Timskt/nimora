# Nimora 运行与故障恢复手册

> 版本：0.1.0-draft  
> 更新日期：2026-07-17  
> 面向：开发、测试、支持人员和高级用户

## 1. 通用处理顺序

1. 记录应用版本、平台、发生时间和 Trace ID。
2. 在诊断中心查看组件健康和最近错误。
3. 停止受影响的扩展、连接器或 Agent 任务。
4. 影响范围不明确时进入安全模式。
5. 生成脱敏诊断包，不直接复制密钥或用户正文。
6. 优先恢复服务，再进行根因分析。

## 2. 应用无法启动

### 检查

- 安装包签名和系统版本。
- 数据目录权限和剩余磁盘空间。
- 最近一次升级或迁移状态。
- Core 启动日志和数据库完整性。

### 恢复

1. 使用 `--safe-mode` 启动，跳过第三方扩展和外联。
2. 若成功，逐项启用最近更新的扩展或资源。
3. 若迁移失败，使用迁移前快照进入上一稳定版本。
4. 若数据库损坏，先复制原文件，再运行只读检查和恢复工具。

不得直接删除整个数据目录作为首选方案。

## 3. 宠物不可见或无法交互

- 使用托盘执行“定位到主屏”和“关闭点击穿透”。
- 检查显示器拓扑、DPI 和缩放记录。
- 切换到官方默认 Character，排除命中区或资源问题。
- 重载 WebView；Core 状态应保持。
- 仍失败时重置窗口布局，不重置用户全部配置。

## 4. CPU 或内存异常

1. 查看 Renderer、扩展、事件和 Connector 指标。
2. 暂停不可见动画和第三方扩展。
3. 检查事件风暴、纹理预算、无限重连和日志增长。
4. 保存 60 秒性能诊断，再终止异常组件。
5. 若 Core 本身持续异常，进入安全模式并重启。

## 5. 扩展连续崩溃

- Supervisor 自动进入 quarantine。
- 保留崩溃摘要、版本、权限和最后 Trace。
- 用户可以回滚上一版本、禁用或清除扩展缓存。
- 配置迁移失败时恢复升级前配置。
- 不允许通过无限自动重启掩盖问题。

## 6. 连接器持续失败

- 区分 DNS、连接、TLS、鉴权、限流、目标策略和远端错误。
- 查看熔断状态、最近成功时间和脱敏请求预览。
- 使用测试事件验证，不重放 Restricted 正文。
- 修改目标或权限后重新确认 destination policy。
- 大量积压时按过期策略丢弃，不能阻塞 Event Bus。

## 7. Gateway 暴露风险

1. 立即进入安全模式或关闭 Gateway。
2. 撤销全部 Token 和现有连接。
3. 检查监听地址、最近客户端和访问审计。
4. 若 Token 可能泄露，轮换相关外部密钥。
5. 恢复时仅监听回环地址并重新配对。

## 8. Agent 异常执行

- 取消当前任务并停止 Agent Worker。
- 查看计划、实际 Tool、参数、已完成步骤和回滚结果。
- 撤销本任务使用的临时授权。
- 对不可撤销外发行为明确告知用户。
- 外部内容导致的异常按 Prompt Injection 安全事件处理。

## 9. 配置或迁移损坏

- 禁止覆盖原始损坏文件。
- 验证 Schema、hash、数据库完整性和迁移日志。
- 按“最近快照 → 导出备份 → 默认配置”顺序恢复。
- 修复后重新执行迁移必须得到相同结果。
- 无法可靠恢复时进入只读模式并允许导出用户数据。

## 10. 磁盘空间不足

清理顺序：临时文件 → 可重建缓存 → 旧日志 → 非当前包版本。不得自动删除当前资源、用户导出、配置、记忆和最后可用快照。

## 11. 安全事件升级

以下情况视为高优先级：权限绕过、签名失效、密钥泄露、Registry 篡改、Restricted 数据出站、远程静默执行。立即停止受影响能力、保存脱敏证据、轮换凭据、评估召回并发布安全说明。

## 12. 恢复验证

- Core 可离线启动。
- 默认角色可见并可交互。
- 数据库和配置完整性通过。
- Gateway 和 Connector 保持预期关闭状态。
- 未经用户确认不自动恢复高危扩展。
- 最近一次用户数据快照可读取。
