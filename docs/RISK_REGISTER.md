# Nimora 项目风险登记表

## Creator 复用旧审查或旧批准

- **风险**：UI 或恶意调用方可能用旧审查结果保存已变化草案，或重放同一批准凭证。
- **缓解**：完整草案 canonical SHA-256 绑定、批准与保存时服务端重审、五分钟 TTL、服务端 Pending Store、原子单次消费、摘要不匹配也消费、Safe Mode 清空。
- **剩余边界**：当前 Diff 是相对未安装基线的 `added`；已安装版本升级仍需实现 `removed`、`scope-changed`、依赖与数据迁移 Diff。

> 版本：0.1.0-draft
