# DeskPet 无障碍、国际化与本地化规范

> 版本：0.1.0-draft  
> 更新日期：2026-07-17

## 1. 无障碍目标

控制中心、命令面板、权限和恢复流程必须满足 WCAG 2.2 AA 的适用要求。宠物本体的纯装饰行为可以不进入辅助技术，但所有功能性操作必须有等价入口。

## 2. 键盘与辅助技术

- 所有功能可通过键盘完成。
- 焦点顺序符合视觉顺序，焦点状态始终可见。
- 自定义控件提供正确 role、name、state 和 keyboard behavior。
- 气泡重要操作同步出现在可访问的通知或任务列表。
- 拖拽操作提供菜单或键盘替代方式。

## 3. 动画与感官

- 支持减少动画、禁用闪烁、禁用屏幕震动。
- 声音提示有字幕或视觉替代。
- 信息不只依赖颜色、声音、方向或动作速度。
- 不使用每秒超过安全阈值的闪烁内容。
- 宠物可设置静止模式而不失去功能。

## 4. 缩放与对比度

- UI 在 200% 缩放下保持功能完整。
- 文本对比度和非文本控件对比度达到 AA。
- 角色主题不能降低系统权限、错误和警告可读性。
- 最小点击目标符合桌面可访问性建议。

## 5. 国际化

- UI 文本不在代码中拼接句子。
- 使用 ICU MessageFormat 处理复数、性别和变量。
- 日期、时间、数字、货币和列表使用 locale API。
- 支持文本扩展 30–50%，避免固定宽度按钮。
- 布局预留 RTL 能力，即使首版不发布 RTL locale。
- 快捷键、字体和输入法按平台和 locale 验证。

## 6. Locale 回退

```text
用户选择 locale
 → 区域语言，例如 zh-CN
 → 基础语言，例如 zh
 → 包声明默认语言
 → 平台 en
```

资源包缺失文本时回退，但商店发布前必须满足声明 locale 的完整度阈值。

## 7. 文案与文化适配

- 避免无法翻译的双关作为关键操作说明。
- 提醒不使用羞辱、威胁或文化刻板印象。
- 时间习惯、工作日和节日由 locale/Profile 决定。
- Character 人设可以本地化，但权限和安全含义必须保持一致。

## 8. 测试

- 使用伪本地化测试文本扩展、双字节字符和 RTL。
- 使用屏幕阅读器完成安装、权限、命令、错误恢复和卸载路径。
- 验证 200% 缩放、高对比度、减少动画和仅键盘操作。
- 资源包的 locale、字体和字幕进入发布校验。
