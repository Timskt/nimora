# ADR-009：桌宠位移采用弹簧-阻尼模型而非线性补间

> 状态：Accepted
> 日期：2026-07-22

## 背景

自主漫游（roam）此前用固定十二帧线性插值移动原生宠物窗口：
`value = start + delta * frame / frames`。线性补间意味着恒定速度、瞬时启动、瞬时停止——读起来像机械滑动，而非有生命的移动。产品原则「先是生命体，再是工具」要求位移具有加速、减速与（欠阻尼时）回弹感。

移动发生在 Rust 宿主侧：宿主循环逐帧调用 Tauri `set_position` 推动无边框透明窗口，而不是在渲染层做 CSS/WebGL 动画。因此运动模型必须落在宿主可测试的纯逻辑里，且不得引入 Tauri、渲染器或 IPC 依赖，以通过架构边界门禁（`scripts/check-architecture-boundaries.mjs`）。

## 决策

新增 Tauri-free 的 `apps/desktop/src-tauri/src/pet_physics.rs`，实现一维弹簧-阻尼振子 `Spring`：

- 采用半隐式（辛）欧拉积分，先更新速度再用新速度更新位置，对振子远比显式欧拉稳定。
- 每个可见帧内部再细分为不超过 `1/240 s` 的整数子步，使刚度较大的弹簧在粗帧率下仍数值稳定。
- 所有进入积分器的值（位置、目标、系数、`dt`）都经 `sanitize` 过滤；`NaN`/`Infinity` 坍缩为 `0`，非正或非有限 `dt` 为空操作，因此错乱的时钟或系数不能把宠物瞬移或把非有限坐标注入原生窗口。
- 漫游默认使用临界阻尼（`damping = 2*sqrt(stiffness)`）：不越过上游已算好的安全边界，同时平滑缓入缓出。`Spring::new` 允许更小阻尼以获得欠阻尼回弹的俏皮手感，供后续动作复用。

漫游循环改为每轴一个 `Spring`，逐帧 `advance` 后 `set_position`；`is_settled`（位置与速度双阈值）判定到达后，精确落在规划目标上，使下游持久化与 Surface 分类看到的是意图坐标而非亚像素残差。保留原有逐帧门禁：Reduced Motion、拖拽、Safe Mode 与非 Walking 状态仍立即中止移动。帧数上限 `PET_WANDER_MAX_FRAMES` 保证不收敛时循环仍终止。

## 备选方案

- **保留线性补间**：最简单，但恒速滑动正是要消除的机械感，且无法表达急停回弹。
- **预计算缓动曲线（ease-in-out tween）**：能做到缓入缓出，但对固定时长/固定终点的曲线；目标或中途被门禁打断时不能自然接续速度，回弹、追随移动目标（如未来贴近光标）都要另起一套逻辑。弹簧天然携带速度状态，可无缝重定向。
- **解析闭式解**：数学上精确，但分段处理临界/欠/过阻尼较繁琐，且不如子步积分器易于扩展到未来的次级动画（尾巴、耳朵）。

选择数值弹簧是因为它用同一套状态化模型统一了漫游、未来的追随移动与次级动画，且易于隔离测试。

## 结果

- 正面：位移具备加速/减速/可选回弹的生命感；模型纯粹、无宿主依赖、可单元测试；同一 `Spring` 可复用于后续 IK、次级动画与碰撞回弹。
- 代价：每帧多出常数级算术与子步循环；引入需治理的调参常量（刚度、到达阈值、帧上限）。
- 缓解：子步上限与 `sanitize` 保证数值稳定与失败安全；调参常量集中在漫游常量区并带文档；行为由 8 项单元测试锁定（临界/欠/过阻尼、非有限输入、粗帧稳定、零刚度静止）。

## 后续

- 次级动画（尾巴/耳朵的 Verlet 延迟跟随）与撞墙压缩-回弹-晕眩仍需在此基础上分别建立可测试语义。
- 欠阻尼手感的具体动作绑定（如玩耍蹦跳）应复用 `Spring::new` 的低阻尼构造，避免各自造轮子。
