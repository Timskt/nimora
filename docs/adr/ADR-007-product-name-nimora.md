# ADR-007：正式品牌与技术命名空间采用 Nimora

> 状态：Accepted
> 日期：2026-07-17

## 背景

早期工作目录使用 `gptpet`，文档草案先后使用 DeskPet 与 AsterPet。`gptpet` 将产品错误地限定为 GPT 应用，DeskPet 描述性过强，AsterPet 又持续强调 “Pet”，无法完整覆盖桌面生命体、自动化运行时、用户代码、角色资产和开发者生态。

## 决策

正式品牌采用 **Nimora（中文名：灵栖）**。名称为自造词，不绑定某种模型、角色形态或用户群体；“灵栖”表达数字生命在用户设备中安全栖居的产品意象。

GitHub 仓库、发布产物和新公共命名空间同步采用 Nimora。公共 Schema 使用 `nimora.*`，JavaScript 包使用 `@nimora/*`，Rust crate 使用 `nimora-*`，桌面应用标识使用 `dev.nimora.*`。本地旧目录名不构成产品标识。

## 结果

- 新代码、文档、包、应用标识和 UI 统一使用 Nimora；中文传播场景可使用“灵栖”。
- 已经发布的公共标识不得静默重命名；当前尚未发布，可直接建立新命名空间。
- 正式公开发布前必须完成商标、域名、软件商店、包注册表与主要社交账号的可用性核验。
- 未来若再次更名，必须新增 ADR 并提供协议、包名、应用数据目录和第三方扩展的兼容迁移方案。
