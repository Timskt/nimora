# Nimora 原生桌面宠物体验规范

> 状态：产品与工程基线  
> 更新日期：2026-07-19  
> 目标：实现类似经典 QQ 宠物“生活在桌面上”的持续陪伴体验，而不是把聊天窗口伪装成宠物。

## 自主漫游与角色朝向

自主 Explore 由 Desktop Coordinator 在当前原生 Work Area 内规划目标，并以有界帧数移动桌宠窗口；Renderer 不读取显示器、系统栏或窗口句柄。角色资源以“默认面向右侧”为统一创作基线，表现层根据权威 Autonomy Sequence 在 Walking 期间确定性投影 Left/Right，并在动作结束后回到 Neutral。内置、Sprite、glTF、VRM 与未来 Live2D 必须复用统一角色舞台变换，不能分别维护方向状态，否则会在取消、拖拽、重启和 Renderer 切换时发生漂移。

边缘姿态不是自主生命循环的终点。Perching、Climbing 与 Peeking 可以像 Idle 一样进入离线自主 Observe、Explore 或 Rest；Core 在动作开始时记录受限 `resumeAction`，完成或安静策略抑制后恢复原边缘姿态，用户拖拽和其它显式状态抢占则丢弃恢复点。`resumeAction` 只允许 Perch、Climb、Peek，篡改快照写入 Work、Sleep 或无活动 Intent 的恢复点必须在持久化边界失败关闭。

从边缘开始的 Explore 由 Desktop Coordinator 沿当前 Surface 移动：Top/Bottom 及其 Corner 保持固定纵坐标水平行走，Left/Right 保持固定横坐标上下攀行，Free 继续使用二维漫游。目标始终位于当前显示器 Work Area 与生命体安全边距内；到达轴向端点时确定性反向而不是原地踏步。Corner 归属沿 Drop 契约保持唯一：顶部 Corner 属于 Top，底部 Corner 属于 Bottom。无显示器、拖拽、Safe/Recovery、Profile 静默或状态变化立即停止，不猜测另一块屏幕，也不授予 Renderer、AI、Skill 或用户代码移窗能力。

朝向只改变角色视觉，不镜像气泡、菜单、阴影命中区域或无障碍文本；Reduced Motion 关闭镜像过渡而不是删除方向反馈。旧资产无需迁移，若资产本身默认面向左侧，应在未来版本化资产描述符中显式声明基础朝向，不能依赖文件名或模型内容猜测。

## 桌面表面与边缘栖息

桌宠位置由 Desktop Coordinator 投影为版本化 Surface Semantic：Free、Left、Right、Top、Bottom 以及四个 Corner。分类只基于当前显示器原生 Work Area、窗口物理尺寸和统一安全边距；无显示器、几何读取失败与跨屏过渡不得猜测 Surface。移动期间沿用 Position Revision 防抖，持久成功后才发布稳定变化；拖拽期间不发布，Drop 成功后立即收敛。

Surface 是宿主瞬时环境投影，不进入 Pet Core 快照，不改变 Home，不授予 Renderer 原生能力。统一角色舞台可据此做低幅度贴边表现，气泡、菜单、阴影、拖拽和无障碍阅读方向保持独立。`pet.perch`、`pet.climb` 与 `pet.peek` 已分别成为正式语义动作，资源包和扩展不得以任意 CSS 类或私有动作绕过其取消、回退与 Reduced Motion 规则。

## 1. 产品定义

Nimora 的宠物必须是独立的原生桌面实体：透明无边框、跳过任务栏、可置顶、可拖拽、可穿透、可跨屏停留，并在控制中心关闭时继续存在。聊天、Agent、自动化和用户代码是宠物可调用的能力，不是宠物存在的前提或唯一交互入口。

发布判定必须同时满足：

- 用户不打开聊天窗口，也能观察、触摸、拖动、照料和陪伴宠物。
- 宠物具有连续状态、自主行为和对桌面环境的有限反应，而非循环播放单一待机动画。
- 断网、Provider 不可用或用户禁用 AI 时，基础桌宠体验完整可用。
- 用户可以一键暂停自主活动、恢复交互、静音、隐藏或退出；任何时候都不得妨碍系统安全界面和关键操作。

## 2. 原生窗口模型

桌宠窗口与控制中心必须分离，并由 Tauri/Rust 宿主管理真实桌面能力：

- **宠物窗口**：透明、无边框、无阴影、跳过任务栏、默认非全屏、独立生命周期。
- **控制中心**：设置、AI、Creator、资产、权限、诊断与数据管理，不承担宠物本体展示职责。
- **交互命中区**：根据角色姿态和模型边界动态计算；透明像素不得形成大面积桌面遮挡。
- **穿透恢复**：开启点击穿透后，托盘、全局快捷键或控制中心必须始终能恢复交互。
- **多屏与 DPI**：保存显示器身份、逻辑位置和缩放；显示器消失时将宠物安全夹回主屏可见区域。
- **平台降级**：平台不支持某项窗口能力时必须显式报告，并采用安全、可恢复的替代行为。

## 3. 桌面生命感

宠物运行时采用分层状态机，而不是由 UI 组件直接拼接动作：

```text
Needs → Mood → Intent → Action Plan → Renderer Semantics → Adapter Animation
```

- `Needs`：精力、心情、亲密度、清洁、饱腹、社交意愿；允许用户关闭或简化照料压力。
- `Mood`：平静、开心、好奇、疲惫、委屈、专注等可解释状态，不以惩罚或焦虑绑架留存。
- `Intent`：休息、探索、靠近、玩耍、提醒、庆祝、请求互动。
- `Action Plan`：可取消、有时限、可恢复的动作序列；用户拖拽、隐藏、睡眠或安全模式拥有更高优先级。
- `Renderer Semantics`：统一使用 `pet.idle`、`pet.walk`、`pet.sleep`、`pet.drag`、`pet.interact` 等语义动作。
- `Adapter Animation`：Sprite、Live2D、VRM、glTF Adapter 映射语义动作；缺失动作必须确定性降级。

自主行为至少包括待机变体、眨眼、张望、睡眠、伸懒腰、桌面小范围漫游、边缘停靠、被拖拽反馈、点击反馈、任务完成庆祝和低打扰提醒。行为调度必须有冷却、频率预算、安静时段和“专注模式”。张望必须使用独立 `pet.observe` 语义，不得借用庆祝或点击来伪造生命感；角色包缺失时确定性回退到 `pet.idle`。

当前实现基线已经将调度放入纯 Rust 领域层：由显式时间和持久序列确定性选择观察、探索或休息，动作具有截止时间与冷却，用户状态可抢占且不会被自主结束动作覆盖。Desktop Host 每秒从活动 Profile 动态解析 `proactiveFrequency`：0% 完全关闭，1–100% 采用五档单调延迟与冷却预算；Focus 和 Presentation 会立即结束仍由自主循环控制的动作，用户拖拽等抢占状态不被覆盖，Offline 模式不依赖网络并继续运行。Explore 由 Desktop Host 映射为原生窗口短路径平滑移动，按当前显示器原生 Work Area 和窗口尺寸夹取安全边距，避开系统报告的菜单栏、Dock、任务栏或 Panel，支持负坐标副屏并复用位置防抖持久化；逐帧复验 Drag、Safe Mode 和语义状态。低频可见性守卫会在跨屏时选择最大工作区交集，在分辨率或系统保留区变化时重新夹取，完全离屏时回到主屏工作区，且拖拽期间不抢夺窗口。Renderer 继续消费统一语义状态。该基线不是最终体验完成声明；自动隐藏系统栏、混合 DPI、环境感知和双平台真机门禁仍需补齐。

活动 Profile 还可配置本地安静时段。策略只持久化是否启用及起止分钟，宿主依据操作系统本地时间计算当前分钟，领域层以半开区间支持日内和跨午夜窗口；启用时起止相同属于无效配置。命中时只抑制自主动作和主动提醒，不冻结生命状态，不阻断手动照料，也不依赖网络。时区或夏令时改变后下一次宿主 Tick 即使用新本地时间，不追赶安静期间错过的自主动作。

拖拽释放由 Desktop Host 根据原生物理坐标完成桌边吸附：仅当窗口距离当前显示器 Work Area 安全边界 32px 内时吸附最近边，远离边缘保持用户位置；负坐标副屏、窗口尺寸、系统保留区和上下左右安全边距均参与计算。`edgeSnap` 是向后兼容的 Profile 策略，默认开启，用户可选择自由摆放。Renderer、Program、Skill 与 AI 不能提交伪造的显示器几何或绕过宿主边界。Tauri Work Area 在 macOS、Windows 和 Linux 平台由原生 Runtime 提供；自动隐藏栏、动态 Dock 位置和不同 Wayland Compositor 的准确性继续纳入真机门禁。

生命状态同样必须完全本地运行。当前基线每十分钟确定性推进 Energy、Mood、Satiety 与 Cleanliness：Energy 每周期下降 1，Mood 每 3 周期下降 1，Satiety 每 2 周期下降 1，Cleanliness 每 4 周期下降 1。首次运行只建立时间基线，休眠或退出后的追赶最多计算 24 小时，避免长时间未启动造成瞬时归零；所有值仅表达照料倾向，不死亡、不生病、不倒扣关系，也不发送惩罚性通知。每次点击互动增加 Mood 与 Affinity 并在 100 封顶。时间戳、生命值和标准 `pet.vitals.changed` Event 由同一 Repository 事务原子保存，失败时内存状态和事件总线均不发布。Pet Overlay 与控制中心通过可信宿主事件刷新同一 Snapshot，控制中心不得再展示与真实领域状态无关的演示数字。旧快照缺少 Satiety/Cleanliness 时迁移为 100，避免升级后制造虚假欠照料。

照料采用独立的 `PetCareAction` 语义，而不是把喂食伪装成动画。Feed 恢复 20 Energy、25 Satiety，并小幅增加 Mood/Affinity；Play 消耗 5 Energy、3 Satiety 与 2 Cleanliness，换取较高 Mood/Affinity；Groom 恢复 30 Cleanliness并提升 Mood 与关系；所有结果在 0–100 饱和。照料拥有 30 秒宿主时间冷却、防止误触和脚本刷值，Drag 具有绝对优先级；合法照料可以安全中止自主动作并进入短暂互动反馈。Overlay 与控制中心提供同一三种操作，Safe/Recovery Mode 失败关闭，浏览器预览只模拟 UI，不冒充原生持久化完成。

经典 QQ 宠物式体验还需要“拥有、使用、陪伴反馈”的本地闭环，而不是网页聊天挂件。Pet 快照因此持有版本化随身背包：新用户与升级旧快照各获得一次 `莓果小食 ×3`、`星星球 ×3`、`泡泡皂 ×3` Starter Pack；缺字段迁移与新建语义一致，用尽后保持为空且不会因重启重复发放。莓果恢复饱腹与精力，星星球换取心情和陪伴成长，泡泡皂恢复清洁；每次使用由独立宿主冷却、Drag 优先级和 0–100 有界规则保护，并以 `pet.inventory.use` Command 与 `pet.inventory.used` Event 原子扣减。库存必须稳定排序、标识唯一且数量为 1–999，保存失败时生命值、库存、冷却和 Event 全部不变。

背包资产完全本地拥有、离线可用、不过期、不因关系下降或 Provider 状态被回收。未来奖励系统、Creator 内容包、导入包和可选商店只能通过受审计的 Grant 契约授予物品，不能绕过库存校验、改变使用语义或回收既有资产。AI、Skill、Program 只能经 Capability Gateway 请求“使用已拥有道具”，不能直接改数量；控制中心关闭、AI 禁用或网络断开时桌宠、背包和生命循环仍完整运行。浏览器 Preview 只用于 UI 模拟，原生 Tauri Overlay 才是桌面生命体权威宿主。

背包必须从桌宠本体直接可达，而不能要求用户先打开控制中心。长按或右键打开的轻量菜单显示当前道具总数，并进入同窗口二级背包页；每项呈现稳定图形、名称、核心效果和实时数量，点击后通过同一 `use_pet_item` 宿主命令扣减并留在背包页。Esc 在二级页先返回宠物菜单，再次 Esc 才关闭并把焦点交还桌宠；空背包仍保留无焦虑说明。该交互不扩大 260×300 原生窗口、不创建网页弹窗，也不把 Renderer 变成库存权威。

生命值会影响行为而不制造惩罚性打扰：Energy ≤ 25 时下一次自主行为稳定优先 Rest；Energy 充足但 Mood ≤ 25 时优先温和的 Observe/互动反馈；其余状态继续按持久序列确定性轮换。Energy 优先级高于 Mood，防止疲惫角色仍被迫活跃。Idle 情绪同步派生为 Sleepy、Sad 或 Neutral，但不会覆盖 Drag、Work、Interact 等活动状态。Overlay 只在悬浮/键盘聚焦气泡中表达需求，不发送系统通知、不自动调用 AI、不访问网络；内置角色提供困倦和低落视觉细节，第三方 Renderer 继续消费统一 Sleep/Interact 行为语义。

Sleep 是领域状态而非纯动画。用户主动休息时，每个生命周期恢复 2 Energy，最多 100；Mood、Satiety 与 Cleanliness 仍按低压力节奏自然演化。自主 Rest 的动作时长短于生命周期，因此在一次合法完成时直接恢复 8 Energy；被用户拖拽、Safe/Focus 策略或其它状态中断时不获得完成收益。两条路径均经过原子 Snapshot/Event 持久化，离线追赶受既有 24 小时上限约束。

宠物名称属于持久 `Pet Identity`，不是用户 Profile 名称、角色资源名称或 Renderer 标签。用户可在控制中心关系卡以及桌宠长按/右键菜单中使用内联表单改名；名称在去除首尾空白后必须包含 1–64 个 Unicode scalar values。改名只替换显示身份，不改变 Pet ID，不重置关系、生命值、背包、纪念、位置、行为策略或角色资产。Command `pet.identity.rename`、Event `pet.identity.renamed` 与 Snapshot 在同一应用事务中提交；校验或 Repository 保存失败时名称、事件和所有其它状态均保持不变。

改名必须完全离线，关闭控制中心、禁用 Provider 或断网时仍可从原生 Overlay 完成。两个窗口通过宿主 Snapshot 事件同步，所有可见文案和 ARIA 名称使用持久名称；Esc 从改名页先返回宠物菜单，再次关闭菜单。Safe/Recovery Mode 失败关闭。AI、Agent、Program、Skill 与第三方模型不得直接写 Snapshot，只能在获得明确 Identity Capability 后经 Capability Gateway 请求相同命令，因此未来语音命名、剧情昵称和家庭多宠扩展不会绕过身份权威。

桌宠拥有独立于当前落点的持久 `homePosition`。用户可以从桌宠本体菜单把当前位置设为家，也可以随时回家；移动、拖拽和自主漫游只改变 `position`，不会悄悄重写家。旧快照缺少该字段时，以最后一次持久位置一次性迁移为家，避免升级后突然跳到屏幕原点。设置与返回分别使用 `pet.home.set` / `pet.home.changed` 和 `pet.home.return` / `pet.home.returned` 原子契约，Repository 失败时窗口和 Snapshot 均回滚。

回家由原生宿主根据当前显示器拓扑、窗口尺寸和安全边距校正目标位置：原显示器移除、分辨率改变或 DPI 布局变化时，选择最大重叠显示器，完全无重叠则回到主显示器可见区域。校正后的实际坐标写入当前 `position`，但保留用户原始 `homePosition`，使显示器恢复后仍能回到真正的家。Drag、Safe/Recovery Mode 均失败关闭；断网、关闭 Provider 或控制中心不影响该能力。

Pet Identity 必须贯穿领域快照、原生窗口标题、控制中心、Overlay、系统窗口管理器与辅助技术。启动窗口直接读取持久名称；改名采用“原生标题先应用、Snapshot/Event 后提交”的可逆事务，持久提交失败必须回滚标题，避免桌面系统与应用界面出现两个名字。内置角色资源名只可作为首次创建默认值，不能成为窗口层永久常量。

## 4. 经典桌宠能力

### 4.1 直接互动

- 单击、双击、右击、长按、抚摸轨迹和拖拽采用明确的手势仲裁，避免点击误判为拖拽。
- 抚摸是独立语义而不是连续 Click：按住后在 12px 安全半径内完成至少 32px、160ms 且含方向反转的往返轨迹才成立；向外达到 12px 立即让位于原生拖拽。合法抚摸增加 Mood 4、Affinity 2 与 BondPoints 2，并发布含有界距离、时长和反转次数的 `pet.interaction.stroked` 事件；原始逐点轨迹不持久化。
- 右键或长按打开同一轻量径向菜单。六个高频入口环绕宠物：投喂、玩耍、梳理、背包、回家和更多；改名、设置家、休息与鼠标穿透在同窗口“更多”页完整保留。轮盘不得扩大 260×300 原生窗口，方向键循环、Home/End 跳转、Esc 逐级返回，读屏按稳定 DOM 顺序朗读，不依赖指针命中角度。
- “更多”页也是桌宠到复杂能力的桥梁：聊天、开始任务和设置通过类型化 `ControlCenterDestination` 请求宿主显示、取消最小化、聚焦控制中心，再投递站内导航事件。桌宠不得构造 URL、注入任意页面名或直接依赖 Agent UI；控制中心只接受注册页面。无 Provider 时仍可打开 Agent 工作区并呈现本地状态，设置始终离线可达。
- 拖拽期间角色必须有抓取姿态；释放时进行可访问区域约束和安全落地反馈。
- 桌边吸附、窗口边缘停靠与漫游均可独立关闭，且不得读取其它应用内容。

### 4.2 照料、成长与收藏

- 照料循环包括投喂、清洁、休息、玩耍和陪伴，但默认采用低压力设计。
- 成长以长期陪伴、任务协作和用户主动互动积累，不通过强制签到、倒计时惩罚或付费焦虑驱动。
- 关系采用双轨模型：`Affinity` 保持 0–100，表达当前关系温度；`BondPoints` 以 JSON 安全饱和整数长期单调累计，表达真实陪伴历程。每 50 点派生一级，等级公式属于领域契约，不由 Renderer、AI 或页面临时定义。
- 关系阶段由 Core 从有效 `BondPoints` 确定性派生：`初识(0)`、`熟悉(25)`、`信赖(100)`、`知己(300)`、`长久相伴(1000)`。阶段不是第二份持久状态，导入旧数据、离线恢复和跨版本升级都不会发生字段漂移；宿主通过 `petRelationship` 权威投影同时输出阶段、下一阶段和等级进度。
- 成长界面首先表达关系称谓与共同经历，其次才显示数值；下一阶段只使用“在自然陪伴中逐渐抵达”等非催促文案，不展示连续签到、限时倒计时、失去关系或付费加速。达到最高阶段后仍允许等级与纪念继续增长。
- 旧快照缺少 `BondPoints` 时以现有 `Affinity` 作为有效成长基线；下一次合法互动再增量迁移，避免丢失已有关系或首次双计数。冷却、拖拽拒绝、持久化失败和生命值 Tick 均不得产生陪伴点。
- 角色人格、关系记忆、服装、房间、道具、动作、声音和纪念册均使用版本化数据契约。
- 经济与收藏系统必须本地可用、可导出、可删除；未来接入商店时不得改变离线资产所有权。

### 4.3 桌面环境互动

- 在用户授权后，可感知显示器边界、工作区、前台应用类别、空闲/锁屏、音频播放状态和专注模式。
- 可执行桌边行走、靠近活动窗口边缘、空闲入睡、回到出生点、任务完成庆祝等动作。
- 默认不得读取窗口正文、键盘内容、密码框、剪贴板或屏幕像素；细粒度感知能力必须逐项授权。
- 全屏应用、演示、游戏、屏幕共享、免打扰和系统安全界面出现时自动降低打扰或隐藏。

`presentation` Profile 是环境自动感知接入前也必须可靠工作的显式降扰基线：激活后原生桌宠窗口隐藏且自主互动暂停，控制中心和后台生命状态继续本地运行；以该 Profile 启动时窗口从创建阶段即隐藏，不允许先闪现再消失。切回其它 Profile 时恢复原位置与目标窗口策略。可见性、置顶、穿透和活动 Profile 使用同一可逆事务，任一步失败必须整体回滚；Safe Mode 始终强制可见，系统托盘恢复属于用户明确覆盖。未来全屏、屏幕共享、游戏和系统免打扰 Adapter 必须复用该策略协调器，而不是各自直接调用窗口隐藏。

## 5. AI、Agent 与用户扩展

- 无 AI 时，本地规则引擎负责基础状态、动作选择、照料和离线对话模板。
- 有 AI 时，AI 可以生成对白、建议动作、编排已注册能力或创建候选 Skill，但必须经过 Tool Registry、风险批准和 Capability Gateway。
- Agent 可把任务进度映射为宠物行为，例如思考、工作、等待确认、完成庆祝和失败安慰；不得展示隐藏思维链。
- 用户代码只能调用版本化 Pet SDK，例如读取脱敏状态、请求语义动作、发起气泡消息和注册受控互动；不得直接操作 Tauri 窗口、文件、网络或渲染器对象。
- 第三方角色包只提供数据和声明式映射；可执行逻辑必须作为独立隔离扩展安装和授权。

## 6. 稳定性与资源预算

- 宠物窗口崩溃后由宿主有限次数重建；连续失败进入内置角色与静态安全模式，避免重启风暴。
- 自主行为和动画在后台、锁屏、电池低电量、热压力或 GPU 压力下自动降频。
- 网络和 Provider 请求不得位于动画主循环；失败不得冻结拖拽、菜单或托盘恢复。
- 状态持久化采用防抖、原子写入和版本迁移；进程异常退出后恢复最后稳定状态而非中间动作。
- Renderer Adapter 必须报告加载、帧率、内存、GPU 上下文丢失与降级原因，但不得收集用户桌面内容。

## 7. 无障碍与审美约束

- 提供减少动态、减少闪烁、关闭音效、字幕、键盘操作、高对比命中轮廓和可调角色尺寸。
- 气泡不遮挡角色面部和系统控件；短文本自动消失，长文本转入可滚动但不抢焦点的面板。
- 动作遵循预备、主动作、缓冲、收尾节奏；禁止无意义持续抖动和高频弹跳。
- 不同角色资产必须共享统一交互语义、反馈时序和安全边界，避免换模型后产品行为失真。

## 8. 发布验收门槛

桌宠能力不能仅凭浏览器预览判定完成。发布前必须在 Windows 与 macOS 真机验证：

1. 控制中心关闭后宠物仍持续运行，重启后位置和状态可恢复。
2. 点击、拖拽、穿透、托盘恢复、置顶、隐藏和退出无不可恢复路径。
3. 多屏插拔、DPI 改变、睡眠唤醒、锁屏、全屏和崩溃恢复行为确定。
4. 离线 24 小时仍能完成基础互动、照料、自主行为和本地持久化。
5. 连续 8 小时运行不存在明显内存增长、GPU 资源泄漏、CPU 空转或通知风暴。
6. Sprite、VRM、glTF 与未来 Live2D 角色对缺失动作采用相同语义降级。
7. AI、脚本、Skill 和资产异常都不能破坏宠物窗口基本交互或越过授权边界。

## Profile 级照料强度

Nimora 的桌宠定位是经典 QQ 宠物式“常驻桌面的生命体”，但不复制强制签到和惩罚性留存。每个 Profile 都可独立选择 `full`、`simple` 或 `off`：完整模式演化 Energy、Mood、Satiety、Cleanliness；简化模式只演化 Energy 与 Mood；关闭模式暂停全部时间驱动的恢复和衰减。三档都保留喂食、玩耍、清洁等主动互动，不删除当前数值，也不依赖 AI、Provider、网络或控制中心。

切换到简化或关闭模式后，宿主仍按到期周期原子推进 `lastVitalsUpdateMs`。因此重新启用完整照料时只能从新的时间基线继续，不能追赶用户明确关闭期间的变化。旧 Profile 缺少该字段时解析为低压力完整照料；Safe/Recovery Mode 继续失败关闭。该策略属于 Profile 配置，Renderer、Program、Skill、Connector 和普通宠物照料 Tool 均不能绕过 Profile 管理能力隐式修改。

## 陪伴纪念收藏

收藏系统的第一条生产纵切是完全本地的陪伴纪念，而不是在线货币或商店。`PetKeepsake` 使用稳定的领域标识，当前在 1、25、50、100 陪伴点依次解锁第一次回应、温柔照料、可信伙伴和百刻相伴。一次互动跨越多个阈值时应一次补齐所有已满足纪念，集合按领域顺序排序且禁止重复；Renderer 只负责把稳定标识映射为本地化名称和视觉。

纪念与 Pet、关系点和对应 Event 使用同一原子快照持久化；Repository 失败时不能出现“界面已解锁但重启后消失”。旧快照缺字段时迁移为空集合，下一次合法互动根据有效陪伴点补齐，不篡改历史数据。纪念无到期、无付费赎回、可随本地数据导出和删除；未来服装、房间、道具和商店只能扩展资产来源与使用规则，不能改变用户已经拥有的离线纪念。

## Agent 工作陪伴反馈

Agent 任务通过版本化 `nimora.agent-companion-signal/1` 向桌宠发布思考、运行、等待确认、完成、失败和取消六种有限状态。信号只包含状态、Task ID 与时间戳，不携带提示词、回答、文件内容、工具参数或任意显示文本。思考和运行映射为工作动作，等待确认恢复待机并提醒用户，完成短暂庆祝，失败和取消使用不施压的安抚文案；终态 4.2 秒后恢复 Core 权威生命状态。

该通道属于可丢弃的瞬时表现，不持久化为宠物领域状态。未启用 Agent、Provider 全禁用、断网、控制中心关闭或事件丢失时，桌宠仍完整支持启动、拖动、照料、菜单、自主行为和离线持久化。未来 Auto Mode、Automation 与第三方 Agent 必须复用同一封闭枚举，不得注入任意动画名或任意气泡文本。

## 边缘栖息体验

- 用户把宠物拖到工作区底边或底角后，可在“更多 → 在边缘栖息”触发 `pet.perch`；同一动作也可由用户程序、Automation 和 Agent 经 Capability Gateway 请求。
- 栖息是正式 `Perching` 状态。内置角色降低重心并收敛呼吸幅度；第三方 Sprite/glTF/VRM 共享同一动作名称与回退规则。
- Bottom Surface 提供栖息锚点，左右 Surface 对应 `pet.climb`，Top Surface 对应 `pet.peek`，避免一个动作承担互相冲突的空间语义。
- 不支持该动作的旧资源包继续显示 `pet.idle`，不会白屏、冻结或执行任意模型表达式；Reduced Motion 下只保留静态姿态。
- 栖息不赋予 Renderer 移窗权限。原生窗口坐标仍只由 Desktop Coordinator 读取、约束、持久化和执行。

## 侧边攀爬体验

- 用户把宠物拖到工作区左侧或右侧后，可在“更多 → 沿侧边攀爬”触发 `pet.climb`；用户程序、Automation 和 Agent 复用同一 Action Catalog 与 Capability Gateway。
- 攀爬是正式 `Climbing` 状态。左、右 Surface 使用相反的支点和轻微贴墙倾角，角落继承对应侧边；Free 与 Top 不伪装攀爬或探头。
- 内置角色使用短周期上下攀爬反馈；Sprite/glTF 优先播放资源声明的 `pet.climb`，缺失时沿 Manifest 回退到 `pet.idle`；VRM 只使用低权重 `surprised` Preset。
- Reduced Motion 下停止上下循环，但保留侧边倾角和贴墙支点；气泡、菜单、阴影、拖拽命中区与无障碍文本保持独立。
- 攀爬动作不读取桌面内容，也不直接移动原生窗口。跨屏、任务栏/Dock 和工作区边界仍由 Desktop Coordinator 的 Work Area 规则约束。

## 顶部探头体验

- 用户把宠物拖到工作区顶部或顶部角落后，可从“更多 → 从顶部探头”触发 `pet.peek`；用户程序、Automation 和 Agent 通过共同 Action Catalog 请求相同语义。
- 探头是正式 `Peeking` 状态。Top、TopLeft、TopRight 使用靠近上缘的支点与小幅探出反馈；Left、Right、Bottom 和 Free 不套用顶部位移。
- 内置角色表现为好奇睁眼和缓慢探出；Sprite/glTF 优先使用 `pet.peek`，旧资源缺失时确定性回退 `pet.idle`；VRM 只使用白名单 `surprised` Preset。
- Reduced Motion 下停止循环探出，只保留静态上缘位置和好奇表情；菜单、气泡、阴影、点击与拖拽命中区域保持原位。
- 探头不允许角色越过 Work Area，也不授予 Renderer 移窗或桌面采集能力；Desktop Coordinator 继续负责原生几何和窗口事务。

## 拖放后的自动边缘收敛

- 用户完成拖拽时无需再次打开菜单：Desktop Coordinator 在最终吸附坐标上分类 Surface，并确定性映射为 Bottom/Bottom Corner → `pet.perch`、Left/Right → `pet.climb`、Top/Top Corner → `pet.peek`、Free → `pet.idle`。
- Corner 只选择一个动作：顶部 Corner 优先探头，底部 Corner 优先栖息；不会同时播放攀爬与探头/栖息，也不会由 Renderer 猜测动作。
- 最终坐标、Pet State、Emotion、命令和领域事件在 Runtime 的同一次持久更新中提交。Core 只接受 Idle、Perch、Climb、Peek 四种被动落点动作，拒绝 Celebrate、Work、Sleep 等主动行为。
- 若持久化失败，Pet 继续处于 Dragged，Surface 事件不发布；若原生窗口已先发生边缘吸附，Desktop Coordinator 将其补偿回拖放前位置，避免原生窗口与领域快照分裂。
- 无当前显示器时收敛为 Idle，而不是猜测主屏边缘；原生位置、尺寸或显示器查询失败显式返回错误，不以零尺寸继续分类。
