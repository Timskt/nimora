## 三、核心设计哲学（从你想法提炼 + 工程增强）

1. **平台优于玩具**：宠物是身体；事件总线 / 技能 / 连接器 / Agent 是骨架。  
2. **一切皆注册**：命令、菜单、动作、触发器、工具、Widget 都注册到 Registry，UI 只展示贡献点。  
3. **官方功能即技能**：番茄钟、提醒等也用 Skill 实现，逼 API 稳定（狗粮化）。  
4. **默认安全**：开放默认关；出站默认本机；高危默认确认；全程可审计可撤销。  
5. **统一事件信封**：`deskpet.event/1` 统一内外；协议只是 transport。  
6. **崩溃隔离 + 配额**：技能/Agent 不拖死主进程。  
7. **渐进交付、架构不欠债**：可分期，边界第一天按终态预留。  
8. **无 AI 也能用核心**：离线陪伴 + 规则自动化永远可用；AI 是增强层。

## 四、你提过的需求总表（优化归并）

### 4.1 基础桌宠（最初诉求）
- 透明无边框、置顶、点击穿透、拖拽、缩放、多屏、DPI  
- 状态机行为：idle / walk / sleep / 点击反馈 / 拖拽姿势  
- 托盘、右键菜单、设置、开机自启、本地配置  
- 皮肤、音效、轻度养成（喂食/抚摸/数值）  
- 休息/喝水等健康提醒（后并入自动化）

### 4.2 高自定义（你第二次强调）
- 外观：皮肤 / Live2D 或序列帧 / 气泡 / 主题  
- 行为：权重、移动范围、自主频率、性格钩子  
- 交互：点击/拖拽/手势语义可绑  
- 菜单与托盘可扩展  
- 快捷键、Profile（工作/摸鱼/夜间）  
- 配置导入导出、YAML/JSON 高级编辑、热重载  
- 分层体验：L0 点选 → L3 写代码

### 4.3 定时 · 自动化 · 提醒 · 快捷指令
- 统一模型：`Trigger → Conditions → Actions → Cooldown`  
- 触发：定时、cron、宠物事件、系统事件、热键、命令、webhook  
- 条件：时段、工作日、Profile、全屏、状态、随机  
- 动作：说话、动画、通知、打开应用/URL、切 Profile、跑技能/脚本  
- 提醒模板：喝水、久坐、眼药、会议、下班；贪睡/打卡/勿扰  
- 编辑器：表单 + 流程图 + YAML  
- 命令面板（类 Spotlight / VS Code）：搜索执行一切

### 4.4 可插拔技能仓库（你的护城河直觉）
- Skill = 可安装功能单元：命令、菜单、自动化扩展、Widget、行为钩子、设置页  
- 权限沙箱、热插拔、崩溃隔离、配额  
- 用户脚本（轻量）+ 完整 Skill SDK/CLI  
- 仓库：本地装、URL/文件、官方源、私有源、Git（规划）  
- 发包多类型：skill / skin / flow / profile / agent / bundle  
- 深度链接一键安装：`deskpet://install?...`  
- 信任分级：Official / Verified / Community / Local

### 4.5 对外开放与「发包」（你第三次强调）
- 第三方不只装皮肤，还要：**装能力、调能力、读数据、被回调**  
- 本地 Gateway：REST + WebSocket + SSE（入站）  
- Token + Scope 鉴权、配对授权（宠物弹确认）  
- Client SDK（JS/Python）、CLI  
- 包规范 + 多 Registry + 签名更新  
- 厂商路径：装 Bundle → 授权 → 推数据/控宠

### 4.6 主动出站数据（你明确要做全）
- **不只等人连你，你要主动往外推**  
- 协议矩阵（增强定案）：  
  | 协议 | 方向 | 用途 |  
  |------|------|------|  
  | HTTP Webhook | 出站 | 业务后端、自动化中台 |  
  | WebSocket Client | 出站 | 实时网关、仪表盘 |  
  | SSE Client | 出站 | 订阅上游再驱动宠物 |  
  | UDP | 出站 | 灯效、OBS、局域网低延迟 |  
  | REST / WS / SSE | 入站 | 第三方查询/控制/听流 |  
- 统一 Connector 配置：过滤、模板、重试、限流、审计  
- 安全：目标白名单、密钥 SecureStore、默认 127.0.0.1、一键断网  
- 不做：无审计裸 socket；UDP 传 Token/聊天全文

### 4.7 第三方 AI Agent（你要「更帅气」）
- 配置多家云端/本地模型（OpenAI 兼容 / Anthropic / 自定义 base_url / Ollama）  
- **BYOK**：自备 Key，不绑死厂商  
- Agent = 编排层：自然语言 → 规划 → 调已有 Tools（命令/技能/自动化/连接器）  
- 默认半自动确认；高危必确认  
- 无 Key 降级：规则意图匹配  
- Agent Pack 人设包可切换  
- 记忆、主动建议（频率受限）  
- 对外 `/v1/agent/chat|run`  
- **明确不做**：独立套壳聊天当主产品；Agent 权限大于用户

## 六、工程定案（最像完整项目、不是过家家）

| 决策点 | 定案 | 理由 |
|--------|------|------|
| 客户端壳 | Tauri 2（Rust） | 常驻轻、系统能力强 |
| 渲染 | PixiJS 序列帧 MVP → Live2D 可演进 | 先稳性能 |
| 业务核心 | Rust Core + TS UI/Skill | 稳 + 生态 |
| 技能运行 | 独立 Skill Host 进程 | 崩溃隔离 |
| 事件 | 统一 `deskpet.event/1` | 生态契约 |
| 出站 | Connector 框架，禁止技能乱写裸网 | 可审计 |
| 开放默认 | Gateway 默认关，首次启用 | 安全默认 |
| AI | BYOK + tool-calling + 确认执行 | 有用可控 |
| 官方功能 | 全 Skill 化 | API 真稳定 |
| 配置 | 用户 YAML / 内部 Schema 校验 | 可读可验 |
| 分期 | M0→M9 见 PRD | 可发布切片 |

**仓库逻辑结构：**  
`apps/`（desktop/web-demo）· `packages/`（core, gateway, connectors, automation, agent, skill-api, client-js, schemas, cli）· `skills/official/` · `docs/` · `tests/`

## 九、安全与边界（你没全说、但产品必须加）

- 默认本机；局域网暴露 = 危险开关 + 红字  
- 出站目标白名单；密钥 SecureStore；载荷脱敏  
- Agent 不大于用户权限；高危工具强制确认  
- 安全模式：停 Gateway + 停全部出站  
- 明确不做：默认键鼠钩子偷隐私、无签名进官方源、UDP 传机密、任意 Shell 插件
