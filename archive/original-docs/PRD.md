## 1. 产品概述

| 项 | 内容 |
|---|---|
| 产品名称 | DeskPet Platform（桌宠平台） |
| 定位 | 本地常驻的桌面生命体运行时 + 可插拔技能平台 + 本地开放网关 + 可审计数据出站平面 + 可调用工具的 AI Agent |
| 目标用户 | 办公族、学生、萌宠爱好者；进阶：自动化玩家；极客：技能开发者；厂商：第三方软件对接 |
| 核心价值 | 陪伴感、轻度解压、个性化、可编程、可联动一切 |
| 形态 | 桌面客户端（Windows 10/11、macOS 12+）；本期以 Web+Node 演示核心，终态 Tauri 2 |
| 商业模式 | 免费 Core + 官方基础技能；技能商店付费/打赏；Pro 云同步/多宠/私有源；企业静默部署 |

### 1.1 产品支柱（5 根）
1. **Pet Core**：渲染、行为、互动、养成
2. **Customization Layer**：外观/行为/UI/键位/Profile 高度可配
3. **Automation Engine**：定时、触发器、条件、动作流水线
4. **Skill System（可插拔）**：技能包 = 功能插件，热插拔
5. **Open Platform + AI**：对外发包、数据出站、第三方 AI Agent

## 5. 约束与假设
- 不做手机端（本期桌面）
- 不做完整 3D 开放世界
- 不读用户文件内容训练
- 默认角色版权需自有/可商用
- mac 置顶全屏空间与 Win 差异分别验收
- 默认禁用全局键鼠 hook
- UDP 不传 Token/对话原文

## 9. 待拍板决策（产品级）
1. 技能语言主推 TS（一致）
2. 壳 Tauri 2（轻）
3. 沙箱：敏感技能子进程（推荐）
4. 官方功能全 Skill 化（是）
5. 商店是否 v1 上线（先本地+URL，架构按商店）
6. 系统自动化默认紧，开发者松
7. 配置用户 YAML 内部 JSON
8. 默认 Local Gateway 关闭，首次启用
9. Agent 默认确认执行
10. 数据默认仅 presence + 显式授权
